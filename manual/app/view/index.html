What's Liber ?

Liber is a very simple php framework based in MVC pattern, indicated to who want to choose own database layer, template engine and so on.

Get Started

Before start, you need understand some conventions about how this framework works.

In general, it work like a MVC pattern.

The app/ folder

This is a folder where you will put your web application, within it you will find the structure of how Liber applications seems.

  • app/assets/: where are stored any public file used in your application; (optional)
  • app/class/: personal classes that yon can use in your application; (optional)
  • app/config/: store the config.php file, that configure your application or another file that you consider useful to put there;
  • app/controller/: store several controller files used by application;
  • app/helper/: store several helper files used by application, the helper file can't be a PHP class, only simple functions; (optional)
  • app/layout/: store layout files that are simple HTML files that replace already existing view files; (optional)
  • app/log/: store several log files wrote by application;
  • app/module/: store several module folders, a module has the same structure that an app has; (optional)
  • app/plugin/: store several plugins files used by application; (optional)
  • app/template/: store several HTML files that build the general visual of application; (optional)
  • app/view/: store several view files used by application; (optional)

The minimal app/ folder structure

  • app/config/
  • app/controller/
  • app/log/

How to start

At first you need load Liber.php class file using:

        // You can choose where you will put Liber framework folder
        // Example including Liber framework path
        include_once "../Liber/Liber.php";

After you need to load the application specifing the app/ folder path:

        // prepares enviroment to Liber application
        Liber::loadApp( realpath('../SomeApp/').'/' );

And to run your application:

        Liber::run();

Running application is when the many things happens, like routing requests, call controllers, views ...

If you want only use Liber framework classes and helpers inside another existing application, you can ignore the Liber::run() code.

But in this case, you need to do some changes in Liber enviroment that are described as follow:

        // Liber don't know where your application URL start
        // setup APP_URL to your real URL application base
        Liber::conf('APP_URL', "http://www.yourapplication.com/");

Others Code Examples

        // get configuration data
        echo Liber::conf('LOG_PATH');
        echo Liber::conf('APP_ROOT');

        // set your own configuration value
        // warning, see on API docs which are the keywords to avoid conflicts or strange behaviour
        Liber::conf('MY_PERSONAL_CONFIG', 'value');

        // load a class (framework)
        Liber::loadClass('Mailer');
        $m = new Mailer;

        // load and return a object class
        $m = Liber::loadClass('Mailer', true);


        // load and return many object classes
        list($m, $cart) = Liber::loadClass( Array('Mailer', 'Cart'), true);
